Hexo折腾记录

发表于 2017-10-17 | 更新于 2017-10-19 | 分类于 折腾记录 | 评论数： | 阅读次数：

前言

几年前被人建议用Hexo博客程序，我以麻烦和功能少的原因拒绝了。

而后我从WordPress换成了Typecho，一直到两个月前，Typecho还在服役中。这之后服务器断了，我也没腾出时间折腾。

但最根本的原因还是我大概有一年没有更新博客了，更没有余力去管博客是否还能正常访问。

直到昨天，我偶然看到一个UI不错的博客，恰好就是Hexo，突然想起自己的博客还没有修复，不如搭建个Hexo好了。

于是怀着一劳永逸的目标，我开始了搭建。

Hexo的安装

建议参考官方文档操作，因为官方文档说的已经相当明白且拥有多种语言包括中文，并不再阐述太多，

总结下便是

  1. 本地部署hexo
  2. github建立新的以“github用户名.github.io”为名称的repository
  3. 本地编译好hexo（hexo -g
  4. 部署至Github（hexo -d
  5. 访问“github用户名.github.io” 即可

我的Hexo个性化操作

1.安装主题

这里我安装的NexT主题，是目前我见过的最完善的主题，提供四种UI布局和无数的第三方扩展，官方文档提供了详细的主题安装教程和第三方服务使用说明，同样不再过多阐述。

2.我的自定义设置

  1. 添加了一系列内置的侧栏选项，包括标签、分类、关于等
  2. 手动添加了留言板页面，并设置了图标为内置Font Awesome图标中的Book
  3. 添加了侧边栏头像和社交链接，并手动设置了steam的图标
  4. 开启了文章末尾的打赏功能（内置插件
  5. 开启了侧栏的友情链接（内置插件
  6. 设置了评论系统为Valine（内置插件，但建议手动替换Panjunwen的完善版，增加了如头像显示，Akismet垃圾评论过滤，后台管理等相当实用的功能。因为完善版作者的教程是基于ghost，所以我们修改\themes\next\layout_third-party\comments下的valine.swig文件中的第二个链接为https://panjunwen.github.io/Valine/dist/Valine.min.js，然后忽略Panjunwen教程中的“云引擎一键部署”之前的所有内容，只做之后的内容即可
  7. 设置了评论系统的邮件通知（通过Leancloud，具体方法见Valine的官方说明
  8. 设置了统计系统为不蒜子（内置插件，内置里最轻量的，适合不需要具体分析数据的用户，具体效果在页面最底部
  9. 添加了侧栏搜索按钮为Local Search（内置插件，同样内置里最轻量的，为本地搜索，弹出ajax窗口进行搜索，具体效果在左侧搜索按键

3.\themes\next\_config.yml

然后是主题配置文件的应用，可以说很多功能需要自行发现，作者都没有介绍，这里我只对我所使用的NexT下的Gemini主题做说明，以下所有修改均在此文件中。

实际上作者很善于注释，所有问题只要英语没问题均能轻松解决。

下面我只说我用到的以及你们比较可能用到的部分

首先确保

左侧栏社交信息只显示图标

icons_only: true

添加一个Steam图标

steam: _422

侧栏显示在左侧

position: left

返回顶部按钮显示在侧栏下方而非右下角

b2t: true

在返回顶部按钮上显示目前为止的百分比

scrollpercent: true

将每个页面滚动到的位置保存在cookies上

save_scroll: false

自动摘录前150个字，之后显示阅读更多，否则可以使用!–more–标签作为分割线

字数统计

wordcount: ture

阅读时长预计

min2read: ture

总字数统计 TotalCount

totalcount: ture

版权所有cc3.0显示

在浏览器宽度特别小的时候，设置margin：0，padding：0

mobile_layout_economy: true

在安卓手机Chrome等浏览器顶栏变色为指定颜色

android_chrome_color: "#222"

一些问题

1.修改本地文件后

修改本地hexo文件，包括添加了新的文章、页面，更改了hexo或主题的_config.yml文件，更换了主题等操作后

建议选择性进行下列操作：

清除缓存文件db.json 和已生成的静态文件public（是否需要此操作看是否可看具体情况

$ hexo clean

本地编译，生成静态文件（进行了上一个操作后必须进行此操作

$ hexo generate（=hexo g

此时可以先本地检查下效果（可选，建议进行，以确保没有错误的情况下进行下一部操作

$ hexo server（=hexo s

文件生成后部署网站至Github

$ hexo deploy（=hexo d

完毕。

2.NexT主题第三方服务的选择

  • 评论系统

主题内置了相当多的评论系统，其中disqus和Facebook comments因为墙的原因国内大多无法正常使用，而网易云跟帖和来必力实在是与主题不协调。

其中大多数都是选择通过各种账号登录，即可回复，其中友言是我唯一能接受的，但因本人更喜欢老的那种免登录的评论方式（必须输入昵称，可选择输入邮箱和网址），所以剩下了hypercomments，valine和hashover（非内置）

最终选择valine是因为他的纯粹和漂亮，以及很快的响应速度

  • 数据统计系统

很多年前折腾过百度，谷歌，cnzz这三个分析系统，但他们究竟对一个个人的独立博客有什么用，我始终想不明白。

现在发现内置的不蒜子统计相当纯粹，且仅仅uv和pv在页脚的显示也满足了我的需求，没想太多便用上了。

  • 内容分享服务

最早用过的是友荐，但现在想想这玩意儿似乎也没什么存在必要，对于我这种很少会写经验分享的我来说

  • 搜索服务

一直很讨厌那种通过搜索引擎服务商提供的搜索方式，尤其是那种用谷歌的，故选择了local search这个本地搜索系统，发现搜索速度意外的快，有点手机启动器全局搜索的感觉

总结

这次的折腾没少废了精力，但折腾过后除非github不给服务或是有了更方便的静态页面博客系统，便不再折腾了。从wordpress到typecho再到hexo，折腾多了，也便不想多折腾了。

这次一步到位，之后只维护内容和评论便是。

而之前的一百多篇文章和一千多条评论也不会搬过来了，准备把这当成一个新的开始，重新定义”我的博客“。（同样的事发生在许多年前sina to wordpress的时候）。
